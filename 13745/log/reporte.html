Reporte Final del Proyecto

Conclusion

Durante este trabajo, con mi compañero, pudimos aplicar conocimientos teóricos y prácticos sobre la computación en la nube junto a otros ramos antes vistos, como linux y programación web. Logramos configurar correctamente una máquina virtual en Azure, instalar servicios y desplegar contenido web. El trabajo en equipo y la planificación fueron claves para completar las tareas sin errores significativos.

Resumen de Arquitectura

Utilizamos una arquitectura simple basada en una VM Linux en Azure, configurada como servidor web mediante Apache. El contenido se desplegó desde un repositorio remoto (Github).

Medidas de Seguridad Implementadas

Durante el proceso de despliegue y configuración, se incorporaron diversas medidas orientadas a fortalecer la seguridad del entorno:

  • Se utilizó Azure Bastion para acceder a la máquina virtual de forma segura, evitando la exposición de puertos como RDP o SSH a internet.
  • Se configuraron grupos de seguridad de red (NSG) para definir reglas precisas sobre el tráfico permitido hacia y desde la VM.
  • Las conexiones a servicios externos, como Git, se realizaron mediante protocolos cifrados (HTTPS/SSH), asegurando la integridad y confidencialidad de los datos transmitidos.
  • Antes de proceder con la instalación de software, se actualizaron los paquetes del sistema operativo para reducir vulnerabilidades existentes.

Reflexión Final

Este proyecto nos brindó una visión integral del ciclo de despliegue de una aplicación web en la nube, abarcando desde la creación y configuración de la infraestructura hasta su puesta en marcha y visualización final. El uso de herramientas como Azure, Git y Apache permitió consolidar conocimientos adquiridos en módulos anteriores, especialmente en áreas como automatización, seguridad, gestión de servidores y trabajo colaborativo. En conjunto, esta experiencia reforzó competencias fundamentales para el desarrollo profesional en entornos de tecnologías cloud.
